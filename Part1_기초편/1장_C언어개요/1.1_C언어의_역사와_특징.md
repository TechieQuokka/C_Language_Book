# 1.1 C언어의 역사와 특징

## C언어의 탄생 배경

C언어는 1972년 미국 벨 연구소(Bell Labs)의 데니스 리치(Dennis Ritchie)에 의해 개발되었습니다. 당시 벨 연구소에서는 UNIX 운영체제를 개발하고 있었는데, 기존의 어셈블리 언어로는 운영체제를 효율적으로 작성하기 어려웠습니다. 이러한 문제를 해결하기 위해 고급 언어의 편리함과 저급 언어의 효율성을 모두 갖춘 새로운 프로그래밍 언어가 필요했습니다.

## C언어의 발전 과정

### 1. B언어에서 C언어로
- **B언어 (1969)**: 켄 톰슨(Ken Thompson)이 개발한 C언어의 전신
- **C언어 (1972)**: 데니스 리치가 B언어를 개선하여 개발
- **K&R C (1978)**: 브라이언 커니핸(Brian Kernighan)과 데니스 리치가 공동 저술한 "The C Programming Language" 출간

### 2. 표준화 과정
- **ANSI C (C89/C90)**: 1989년 미국 국가표준협회(ANSI)에서 표준화
- **C99**: 1999년 개정된 표준, 새로운 자료형과 기능 추가
- **C11**: 2011년 개정, 멀티스레딩 지원 강화
- **C18**: 2018년 기술적 수정사항 반영

## C언어의 주요 특징

### 1. 절차지향 프로그래밍 언어
C언어는 절차지향(Procedural Programming) 패러다임을 따릅니다. 프로그램을 함수들의 집합으로 구성하며, 순차적인 실행 흐름을 가집니다.

```c
#include <stdio.h>

// 함수 선언
int add(int a, int b);
void printResult(int result);

int main() {
    int x = 10, y = 20;
    int sum = add(x, y);        // 함수 호출
    printResult(sum);           // 결과 출력
    return 0;
}

// 함수 정의
int add(int a, int b) {
    return a + b;
}

void printResult(int result) {
    printf("결과: %d\n", result);
}
```

### 2. 저급 언어와 고급 언어의 특성을 모두 보유
C언어는 **중급 언어(Middle-level Language)**로 분류됩니다:

- **고급 언어적 특성**: 가독성이 좋고 구조화된 프로그래밍 가능
- **저급 언어적 특성**: 메모리 직접 접근, 하드웨어 제어 가능

```c
// 고급 언어적 특성 - 구조화된 코드
for (int i = 0; i < 10; i++) {
    printf("%d ", i);
}

// 저급 언어적 특성 - 포인터를 통한 메모리 직접 접근
int *ptr = &variable;
*ptr = 100;  // 메모리 주소에 직접 값 저장
```

### 3. 컴파일 언어
C언어는 컴파일 언어로서 다음과 같은 특징을 가집니다:

- **소스 코드 → 기계어 변환**: 컴파일러를 통해 실행 파일 생성
- **빠른 실행 속도**: 컴파일된 기계어로 직접 실행
- **플랫폼 의존적**: 각 운영체제별로 컴파일 필요

```
소스코드(.c) → 전처리기 → 컴파일러 → 어셈블러 → 링커 → 실행파일(.exe)
```

### 4. 간결하고 강력한 문법
C언어는 비교적 적은 수의 키워드(32개)로 구성되어 있으면서도 강력한 기능을 제공합니다:

```c
// 간결한 문법으로 복잡한 작업 수행
int arr[] = {1, 2, 3, 4, 5};
int *p = arr;  // 배열의 첫 번째 원소를 가리키는 포인터
printf("%d\n", *(p + 2));  // 세 번째 원소 출력: 3
```

### 5. 포인터 지원
C언어의 가장 특징적인 기능 중 하나는 **포인터(Pointer)**입니다:

- **메모리 주소 직접 조작**: 효율적인 메모리 관리
- **동적 메모리 할당**: 런타임에 메모리 크기 결정
- **참조에 의한 호출**: 함수 간 효율적인 데이터 전달

```c
// 포인터를 이용한 swap 함수
void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

int main() {
    int x = 10, y = 20;
    swap(&x, &y);  // x와 y의 주소를 전달
    printf("x: %d, y: %d\n", x, y);  // x: 20, y: 10
    return 0;
}
```

### 6. 이식성 (Portability)
표준 C로 작성된 프로그램은 다양한 플랫폼에서 컴파일 가능합니다:

- **ANSI C 표준 준수**: 플랫폼 독립적인 코드 작성
- **조건부 컴파일**: 플랫폼별 차이점 처리 가능

```c
#ifdef _WIN32
    #include <windows.h>  // Windows 전용
#elif __linux__
    #include <unistd.h>   // Linux 전용
#endif
```

## C언어의 영향력

### 1. 운영체제 개발
- **UNIX**: C언어로 재작성되어 이식성 확보
- **Linux**: 커널의 대부분이 C언어로 작성
- **Windows**: 시스템 프로그래밍에 C언어 사용

### 2. 다른 언어에 미친 영향
C언어의 문법과 개념은 많은 후속 언어들에게 영향을 미쳤습니다:

- **C++**: C언어를 확장한 객체지향 언어
- **Java**: C 문법을 기반으로 한 플랫폼 독립적 언어
- **C#**: C와 C++의 영향을 받은 .NET 언어
- **JavaScript**: C 스타일 문법 채택

### 3. 임베디드 시스템
C언어는 임베디드 시스템 개발의 표준 언어로 자리잡았습니다:

- **마이크로컨트롤러 프로그래밍**
- **IoT 디바이스 개발**
- **실시간 시스템 구현**

## C언어를 배워야 하는 이유

### 1. 프로그래밍의 기초 이해
- **메모리 관리**: 직접적인 메모리 제어를 통한 효율성 학습
- **컴퓨터 구조 이해**: 하드웨어와 소프트웨어의 상호작용 학습
- **알고리즘 구현**: 순수한 로직에 집중한 프로그래밍

### 2. 다른 언어 학습의 기반
- **문법적 유사성**: 많은 언어들이 C 스타일 문법 채택
- **개념적 기초**: 변수, 함수, 제어구조 등의 기본 개념 습득

### 3. 실무 활용도
- **시스템 프로그래밍**: 운영체제, 드라이버 개발
- **임베디드 개발**: 하드웨어 제어 프로그램 작성
- **성능 중심 애플리케이션**: 게임 엔진, 과학 계산 프로그램

## 정리

C언어는 1972년 탄생 이후 50년 넘는 시간 동안 프로그래밍 언어계의 중심에 서있는 언어입니다. 간결하면서도 강력한 문법, 뛰어난 이식성, 그리고 시스템 수준의 프로그래밍이 가능한 특성으로 인해 현재까지도 널리 사용되고 있습니다.

C언어를 학습함으로써 프로그래밍의 기본 원리를 깊이 있게 이해할 수 있으며, 이는 다른 언어를 학습하는 데 있어서도 탄탄한 기초가 될 것입니다.
